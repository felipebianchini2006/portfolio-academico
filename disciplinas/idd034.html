[IDD034] Técnicas Avançadas de Banco de Dados Relacional e Não Relacional

Professor: Alexandre Aparecido Bernardes

Sobre a Matéria

Esta disciplina aprofunda os conhecimentos em sistemas de gerenciamento de banco de dados, explorando tanto as nuances dos bancos de dados relacionais avançados quanto a crescente importância dos bancos de dados não relacionais (NoSQL). Abordamos técnicas de otimização de consultas, manipulação de dados complexa e a escolha da tecnologia de banco de dados mais adequada para diferentes cenários, preparando para desafios do mundo real na persistência e gestão de informações.

Tópicos Abordados

Apresentação SSMS e Revisão Comando DDL

Exploração do SQL Server Management Studio (SSMS) como ferramenta principal para administração de bancos de dados SQL Server. Revisão e aprofundamento dos comandos DDL (Data Definition Language) para criação, alteração e exclusão de estruturas de banco de dados como tabelas, índices e esquemas.

Tipo de Junções e Tabelas

Estudo detalhado dos diferentes tipos de JOINs (INNER JOIN, LEFT JOIN, RIGHT JOIN, FULL OUTER JOIN, SELF JOIN) para combinar dados de múltiplas tabelas, compreendendo suas aplicações e resultados.

Subconsultas e Views

Análise de subconsultas (subqueries) para realizar consultas complexas que dependem do resultado de outras consultas. Criação e utilização de Views para simplificar a visualização de dados, encapsular lógica complexa e controlar o acesso a informações.

Views Avançadas

Aprofundamento em conceitos de Views, incluindo Views atualizáveis, com schema binding e particionadas, entendendo como elas podem ser utilizadas para otimização e segurança em cenários mais complexos.

Stored Procedure

Desenvolvimento e utilização de Stored Procedures (procedimentos armazenados) para encapsular blocos de código SQL que podem ser reutilizados, melhorando a performance, segurança e manutenção do banco de dados.

Functions

Criação e aplicação de Functions (funções) no SQL, distinguindo-as das Stored Procedures e utilizando-as para realizar cálculos ou retornar valores escalares ou em forma de tabela dentro de consultas.

DCL

Estudo da DCL (Data Control Language), com foco em comandos GRANT e REVOKE para gerenciar permissões de acesso e segurança de dados no banco de dados, controlando quais usuários podem realizar quais operações.

Curiosidades

  • Bancos de dados NoSQL (Not Only SQL) ganharam destaque com o crescimento de big data e aplicações web escaláveis, sendo flexíveis para dados não estruturados.
  • Stored Procedures e Functions podem reduzir o tráfego de rede e melhorar a performance, pois o código é executado diretamente no servidor do banco de dados.
  • A segurança em bancos de dados é crucial; o DCL é a base para proteger informações sensíveis e garantir a integridade dos dados.
